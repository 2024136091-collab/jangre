# 실패 사례와 원인

## 001 — GitHub PAT 채팅 노출 (2026-04-28)

### 상황
Private 리포지토리 클론을 위해 Personal Access Token 필요

### 시도한 것
채팅창에 토큰을 직접 붙여넣음

### 결론
- 토큰이 대화 기록에 남아 보안 위험 발생
- 즉시 폐기 후 재발급 필요

### 다음에 할 일
토큰은 반드시 별도 터미널 창에서 직접 입력

## 002 — git clone 명령어 오타 (2026-04-28)

### 상황
`{토큰}` 자리에 실제 값을 넣어야 하는데 그대로 입력

### 결론
- 중괄호`{}`는 치환 표시 — 실제 값으로 교체 필요
- URL 인코딩 오류로 인증 실패
