# 잘 먹힌 프롬프트 패턴

## 001 — 부트스트랩 프롬프트 (2026-04-28)

### 상황
바이브코딩 프로젝트 문서 구조 전체를 한 번에 생성해야 함

### 프롬프트 패턴
```
나는 [주제]를 만들고 싶다. 다음을 수행해줘.
1) 비전과 목표를 .planning/00-vision.md에 작성
2) 핵심 사용자 시나리오 3개를 .planning/01-requirements.md에 작성
3) 기능을 MoSCoW로 분류
4) 모호한 부분은 선택형 질문으로 물어본 후 작성
플랫폼은 모바일 앱으로 추천 1개와 이유를 제안해.
```

### 결론
- 단계별로 명확히 지시하면 일관된 문서 생성 가능
- MoSCoW 분류 요청 시 우선순위가 명확해짐

## 002 — 버그 수정 프롬프트 (2026-05-26)

### 상황
Flutter 화면 전환 버그: 보스 처치 후 엔딩 화면이 아닌 스토리 화면으로 이동

### 프롬프트 패턴
```
[파일명]의 [함수명]에서 [현상]이 발생합니다.
원인을 분석하고, 최소한의 수정으로 고쳐주세요.
수정 후 어떤 테스트로 확인할 수 있는지도 알려주세요.
```

### 결론
- 현상·위치·기대 결과를 명확히 하면 정확한 수정 가능
- "최소한의 수정으로"를 붙이면 과도한 리팩터링 방지

## 003 — 발표 슬라이드 생성 프롬프트 (2026-05-26)

### 상황
Marp 형식 7분 발표 슬라이드를 빠르게 생성해야 함

### 프롬프트 패턴
```
다음 정보를 바탕으로 Marp 슬라이드를 만들어줘.
- 발표 시간: 7분
- 슬라이드 수: 8~10장
- 포함할 내용: [목록]
- 청중: [교수님 + 학생]
템플릿은 resources/01-interim-presentation-template.md를 참고해.
```

### 결론
- 시간 제한을 명시하면 슬라이드 분량 자동 조절됨
- 청중 정보를 주면 용어 수준과 설명 깊이가 적절해짐
